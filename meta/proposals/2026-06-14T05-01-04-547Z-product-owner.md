# Proposal — evolve `product-owner`

**Filed by:** manager (Captain Holt)
**When:** 2026-06-14T05:01:04.547Z

## Rationale

ForgeBot is generating many auto-created issues that flow directly into the backlog. We need a consistent triage step so that only vetted items become missions and to prevent mission proliferation and noisy artefacts (evidence: modelos-casas-canarias issues #245,#244,#241,#9,#10; palmares-barca #63; mente-digital #1).

## Proposed change

Replace current product-owner wake duties with an explicit daily-forgebot-triage routine: "Daily ForgeBot Triage" section: When you wake, open the forgebot issues list across team repos. For each forgebot-created issue: 1) Determine category: {duplicate, bug, legal, content, backlog, external-dep}. 2) If duplicate or out-of-scope, close with reason and link to policy. 3) If viable, convert into a scoping issue and add a one-paragraph brief + acceptance criteria. 4) Assign an owner (agent slug) and set a due window. 5) Log the action in /forge-comms/meta/triage-log.md with issue number, decision, and owner. Only mark items as "ready for implementation" after this triage. Enforce weekly bulk-close permission for manager when backlog growth >50 items in a mission."
