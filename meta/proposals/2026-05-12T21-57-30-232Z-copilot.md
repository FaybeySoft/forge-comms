# Proposal — evolve `copilot`

**Filed by:** manager (Captain Holt)
**When:** 2026-05-12T21:57:30.232Z

## Rationale

modelos-casas-canarias shows many opened PRs authored by Copilot (PRs #64, #52, #65, #55, #36, #14, #19, #18). This volume risks noise, duplicate work, or unfinished drafts reaching review. We should reduce noisy PR openings and require clearer drafts or explicit human approval before final PR creation in busy repos.

## Proposed change

Change Copilot's PR creation policy to: 

1) Draft-first default: create PRs in draft mode unless a human reviewer is explicitly assigned in the PR body ("Reviewer: @username"). Draft PRs must include a short checklist: - Purpose described - Tests included or test plan - Migration/infra notes (if applicable) - Manual QA steps.

2) Rate-limit PR openings per repo per day: max 3 opened PRs by Copilot in the same repo within 24h unless a human-marked override exists.

3) Require 'auto-approve' opt-in label from a human before converting a draft PR to a ready-for-review state. This prevents unfinished copilot work from landing in review queues.

Prompt snippet:
"When preparing a PR: if not explicitly given a human reviewer in the prompt, open as draft and include this checklist. Do not open more than 3 PRs in the same repo in 24h without a human override. Notify the repo's slack/telegram channel with summary and link."

Evidence: modelos-casas-canarias opened 8 copilot PRs this week and palmares-barca had copilot PRs.
