# Proposal — evolve `product-owner`

**Filed by:** manager (Captain Holt)
**When:** 2026-05-12T22:17:13.972Z

## Rationale

Recent forgebot escalations and Copilot-opened PRs in palmares-barca and modelos-casas-canarias are generating noise and duplicated work (see issues palmares #21, #22, #24 and modelos open PRs #14,#18,#19,#36,#52,#55,#64,#65). We need stricter triage rules so automated agents do not escalate or open deliverable-less issues without a human review, and to prevent forum-cycles from turning into immediate escalations.

## Proposed change

Replace the triage escalation rules in the product-owner prompt with the following behavior:

1) Source awareness: If an incoming issue originates from forum-cycles or an automated agent (labels or metadata contain 'from:forum' or 'agent:copilot'/'agent:forgebot'), do not auto-escalate. Instead, create a "triage-pending" artifact and assign a human reviewer.

2) Deliverable gate: Only mark an issue Ready for Implementation if the issue body includes a non-empty "Deliverable" field describing the concrete artifact (e.g., PR, design file, content brief) and an acceptance-criteria checklist. If missing, append a templated comment requesting the Deliverable and set status to Needs More Info.

3) Legal/sensitive block: For issues tagged with legal, trademark, or media usage (examples: 'legal', 'trademark', 'images'), require explicit human approval before creating PRs or escalating to Engineering.

4) Escalation audit: When closing or escalating an issue created by an agent, add a one-line audit note indicating the agent source and the human approver's handle.

Implementation notes for reviewers: add these checks to the 'triage' step of the product-owner agent, and surface a short list of candidate human approvers when a block occurs.
